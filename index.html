Newebe

Get the data back home

Growth of web applications has been a huge improvement for social sharing and collaborative work. Unfortunately, to enjoy these new tools, we have traded our privacy. Newebe aims to solve this problem by providing web applications you host at your home and that connect to your contacts.

Installation

Dependencies

  • Python > 2.6
  • Django >= 1.2
  • Couchdb >= 0.11.0
  • Coucdbkit >= 0.4.8
  • Tornado >= 1.2.0
  • Pycurl >= 1.0.0
  • Daemon >= 1.0.0
  • Markdown >= 2.0.0

Procedure

step 1 Install following packages : 

 python python-setuptools python-pycurl python-daemon couchdb git

Via easy_install install couchdbkit, django and tornado : 

 easy_install couchdbkit django tornado markdown

step 2 Get Newebe from Github or unpack the downloaded archive: 

  git clone git://github.com/gelnior/newebe.git

step 3 Ensure database is started, then in Newebe directory execute following command to configure database : 

  python manage.py sync_couchdb

step 4 Update deployment file (deploy/newebe) by changing NEWEBE_PATH and PIDFILE constants to make it point to your Newebe directory (pid file will be created at first startup).

step 4 bis To change database name or port change constants sets in settings.py file : TORNADO_PORT and COUCHDB_DB_NAME. Default port is 8000 and default database name is newebe.

step 5 Copy file deploy/newebe in the folder /etc/init.d/ then give it execution rights. 

  cp deploy/newebe /etc/init.d
  chmod +x /etc/init.d/newebe

step 6 Start Newebe server. 

  /etc/init.d/newebe start

step 7 Check that everything is fine via your web browser : go to the URL http://localhost:port/ (remplace port by TORNADO_PORT value, default is 8000).

Warning

Newebe is in its early stage of development. For now, it is only a prototype without security features. Use it just for fun and don't expect reliability from it.

Developers

Get the code

You can clone the project with Git by running: 

$ git clone git://github.com/gelnior/newebe
Then follow the README instructions to set up your development environment.

Commit policy

Before starting any development, please inform me via the mailing list. It could save you time and will make integration easier. Then fork the code from github, code and make a pull request with comments about what you did.

Please document every code you push or it will be rejected.

What to do

Newebe must stay simple and easy to use. Moreover, there is a lot to do to make it reach its goal. So, if you don't know where to start, focus on big issues that are listed below. They are mandatory to make newebe production ready. Short specifications about what you do are welcome too.

Issues

Here are the main issues to solve :

  • Unit testing
  • Security

  • Note application
  • Document attachment
  • Picture application
  • Back-up

For more details about a subject, send an email to the mailing list and I will answer as soon as I can.
